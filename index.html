Website Deployment Steps

Learn how to deploy your website on GitHub Pages. Follow these straightforward steps to get your site live:

  1. Create or Fork a Repository: Start by creating a new repository on GitHub, or fork an existing one if you're contributing to a project.
  2. Clone Your Repository: Clone the repository to your local machine to start working on your website.
  3. Add Your Website Files: Add your HTML, CSS, JavaScript, and other necessary files to your repository. Ensure your main HTML file is named `index.html`.
  4. Commit and Push Changes: After making your changes, commit them to your local repository. Then, push these changes back to GitHub.
  5. Enable GitHub Pages: In your repository settings on GitHub, navigate to the "Pages" section. Select the branch you want to deploy (usually `main` or `master`), and save your changes.
  6. Access Your Published Site: Once GitHub Pages is enabled and your site is deployed, it will be available at `https://.github.io/`.

Remember, changes pushed to your GitHub repository can take a few minutes to reflect on your live website. If you encounter any issues, check the "Troubleshooting" section for help.

Setting Up a Custom Domain

Customize your GitHub Pages site with a personal domain. Follow these steps:

  1. Purchase a Domain: If you don't already own a domain, purchase one from a domain registrar.
  2. Update DNS Records: Configure the DNS records for your domain. You'll need to set an A record pointing to GitHub's IP addresses, or a CNAME record if you're using a subdomain.
  3. Add Domain to GitHub Repository: In your GitHub repository settings, go to the "Pages" section and add your custom domain in the "Custom domain" field.
  4. Verify Domain Configuration: After adding your domain to GitHub, verify the configuration. It might take some time for DNS changes to propagate.
  5. Enable HTTPS (Optional but Recommended): For security, enable HTTPS on your GitHub Pages site. GitHub provides automated certificates through Let's Encrypt.

Linking a custom domain can give your GitHub Pages site a more professional appearance. Refer to GitHub's documentation for more detailed instructions.
